# QA Playwright Template

This folder contains a Playwright template for the Library Checkout & Return System.

## Default configuration

- Base URL defaults to `http://127.0.0.1:3001`
- The starter spec resets the reserved exam test data in `test.beforeEach(...)` through `POST http://127.0.0.1:3001/api/exam-fixtures/reset`
- Parallel workers default to `1`
- Tests run from the `tests/` directory

## Setup

```bash
npm install
npm run install:browsers
```

Start the companion app from `qa-test-library-management` repository before running Playwright so the local UI and reset endpoint are both reachable.

## Usage

Run against the default local target:

```bash
npm test
```

Override the worker count if you are managing your own test isolation:

```bash
PLAYWRIGHT_WORKERS=4 npm test
```

If your local app is reachable on a different origin, override both values together:

```bash
PLAYWRIGHT_BASE_URL=http://library-management.mekari.io/ PLAYWRIGHT_TEST_DATA_RESET_URL=http://library-management.mekari.io/api/exam-fixtures/reset npm test
```


## Template layout

- `playwright.config.ts`: default runner configuration
- `tests/reset-test-data.ts`: helper used by specs to reset the reserved exam test data
- `tests/library-checkout-return.spec.ts`: starter spec file. Add the test code here.
- `tests/seed.spec.ts`: Playwright planner agent-only scaffold file; do not edit this file

## Implementation notes
- The template assumes the application is already deployed and reachable through `PLAYWRIGHT_BASE_URL` (localhost by default).
